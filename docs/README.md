## 🚀 快速开始

### 我是开发者

**目标**：为项目贡献代码

1. 阅读 [项目设计思路](./开发指南/00-项目设计思路.md)
2. 按照 [快速开始](./开发指南/01-快速开始.md) 搭建环境
3. 跟着 [开发第一个任务](./开发指南/02-开发第一个任务.md) 写代码
4. 参考 [提交贡献](./开发指南/05-提交贡献.md) 提交PR

### 我是用户

**目标**：使用本工具

1. 阅读 [免责声明](./法律声明/免责声明.md) ⚠️ 重要
2. 按照 [快速开始](./用户指南/快速开始.md) 安装
3. 查看 [常见问题](./用户指南/常见问题.md) 解决问题

### 我想了解架构

**目标**：理解项目设计

1. 阅读 [项目设计思路](./开发指南/00-项目设计思路.md)
2. 阅读 [核心架构](./核心文档/核心架构.md)
3. 查看源代码：`任务流程/基础任务框架.py`

## 💡 文档编写规范

### 针对小白开发者

- ✅ 使用简单易懂的语言
- ✅ 提供完整的代码示例
- ✅ 配图说明（截图、流程图）
- ✅ 逐行解释代码
- ✅ 提供调试技巧

### 文档格式

- 使用Markdown格式
- 代码块指定语言：````python`
- 表格对齐
- 链接正确有效

### 示例规范

**好的示例**：
```python
# 获取屏幕图像
屏幕图像 = 上下文.op.获取屏幕图像cv(0, 0, 800, 600)
#                                  ↑  ↑  ↑    ↑
#                                  x  y  宽   高
```

**不好的示例**：
```python
# 截图
img = ctx.op.get(0, 0, 800, 600)
```
