# 07 - 更新日志

## 2026-06-06 - 企业微信通知功能 🆕

### 远程监控机器人状态

**新增文件**：
- `工具包/企业微信通知.py` - 企业微信通知器核心模块

**改动文件**：
- `数据库/任务数据库.py` - 添加 `企业微信webhook` 和 `状态上报间隔分钟` 配置
- `任务流程/基础任务框架.py` - 扩展任务上下文，添加通知方法和定时上报
- `线程/自动化机器人.py` - 集成生命周期通知（启动/停止）
- `主入口.py` - 优化监控中心的死亡通知处理
- `requirements.txt` - 添加 `requests==2.31.0` 依赖

**功能特性**：
- ✅ **启动通知**：机器人启动时自动推送（含截图）
- ✅ **停止通知**：机器人停止时自动推送（含截图）
- ✅ **异常通知**：任务异常时自动推送（含异常信息和截图）
- ✅ **定时上报**：按配置间隔定时推送状态（含截图）
- ✅ **线程安全**：使用线程池异步发送，不阻塞主流程
- ✅ **智能延时**：集成在 `脚本延时()` 中，自动检测上报时机

**技术亮点**：
- 异步发送机制，避免阻塞机器人运行
- 基于时间间隔而非循环次数检查，避免短延时导致过度检查
- 图片自动压缩（JPEG 80%质量）
- 频率限制（最多每秒检查一次）
- 失败容错（通知失败不影响机器人运行）

**架构优化**：
- 重命名：`发送重启请求()` → `发送死亡通知()`（语义更准确）
- 生命周期管理：机器人线程完全自治，监控中心被动等待
- 停止通知时机：在 `finally` 块发送（op 清理前）

**配置说明**：
- `企业微信webhook`：留空则不发送通知
- `状态上报间隔分钟`：0 表示禁用定时上报，推荐30分钟

**使用文档**：
- 用户指南：[企业微信通知配置](../用户指南/企业微信通知.md)
- 开发文档：[添加配置选项 - 实战案例](../开发指南/04-添加配置选项.md#实战案例企业微信通知配置)

**代码精简**：
- 消除 45 行重复代码（原本在三个循环中分别实现定时上报）
- `自动化机器人.py` 从 374 行减少到 333 行（-11%）

## 2026-02 - 天鹰火炮成就刷取功能

**改动文件**：
- 新增 `任务流程/天鹰火炮成就/` 模块
- `数据库/任务数据库.py` - 新增字段 `是否刷天鹰火炮`

**功能说明**：
- 使用YOLO v8 + ONNX Runtime检测天鹰火炮
- 雷电法术攻击11次，±5像素抖动
- 支持GPU/CPU自动切换
- 小目标过滤（宽≥20, 高≥20, 面积≥400）

**技术特点**：
- 独立YOLO v8检测器（与主项目YOLO v5隔离）
- 自动模型转换（PT→ONNX）
- 置信度阈值：0.20

**使用方法**：
- GUI：配置管理中开启"是否刷天鹰火炮"
- 命令行：`python 主入口.py 是否刷天鹰火炮`
