## 公告横幅与用户统计

**位置**：`界面/公告面板.py`、`工具包/公告管理.py`、`.github/workflows/统计.yml`

### 功能说明

主窗口顶部显示公告横幅，开发者通过编辑仓库根目录的 `公告.json` 发布公告，无需服务器。横幅右侧可显示用户统计（仅开发者可见）。

### 数据流

```
[开发者编辑] 公告.json ──────────┐
[Actions 每日执行] 统计数据.json ─┤ 均存放在 GitHub 仓库 main 分支
                                  ▼
客户端启动 → 异步刷新公告与统计()（daemon 线程，三源 fallback）
          → 写本地缓存 公告缓存.json / 统计缓存.json
          → 公告面板先秒显旧缓存，再通过 after 轮询刷新（tkinter 线程安全）
```

### 核心方法

- `公告管理.异步刷新公告与统计()` - 后台线程拉取远程数据写缓存（启动时调用）
- `公告管理.取最新公告(数据)` - 取 id 最大的公告用于横幅显示
- `公告管理.是否开发者模式()` - 检查根目录是否存在 `开发者标记.txt`
- `公告面板._打开公告详情窗口()` - 点击横幅弹窗显示全部历史公告

### 多源容灾

依次尝试三个数据源（应对大陆网络环境），全部失败时沿用旧缓存：
1. `raw.githubusercontent.com`（数据最新）
2. `cdn.jsdelivr.net`（CDN 缓存约 12 小时）
3. `fastly.jsdelivr.net`

### 发布公告（开发者）

编辑仓库根目录 `公告.json`，在 `公告列表` 追加一条并提交推送：

```json
{
  "格式版本": 1,
  "公告列表": [
    {"id": 2, "标题": "标题", "内容": "正文\n支持多行", "日期": "2026-07-04", "重要": true}
  ]
}
```

- `id` 单调递增，横幅显示 id 最大的一条，弹窗按 id 降序展示全部
- `重要: true` 时横幅标题红色加粗
- JSON 写坏或格式版本不识别时客户端静默隐藏横幅，不报错

### 用户统计（仅开发者可见）

- GitHub Actions（`.github/workflows/统计.yml`）每天北京时间 09:23 调用 Traffic API，将克隆数据累积合并到 `统计数据.json` 并自动提交（Traffic API 只保留 14 天数据，定时任务将其永久化）
- 合并按日期幂等覆盖，重复运行不会重复累加
- 用户数使用 14 天窗口去重值（`近14天独立克隆`），口径准确
- 在项目根目录创建 `开发者标记.txt`（内容随意，已被 gitignore）后，横幅右侧显示"👥 近14天用户 N · 累计拉取 M 次"；普通用户无此文件，只看到公告，也不会下载统计数据
- Traffic API 需要 push 权限：若 workflow 报 403，创建 PAT（classic 勾 `repo` 或 fine-grained 授予 Administration: Read-only）存入仓库 Secrets 命名 `TRAFFIC_TOKEN`

## 天鹰火炮成就刷取

**位置**：`任务流程/天鹰火炮成就/`

### 功能说明

自动搜索敌人村庄中的天鹰火炮，使用雷电法术进行11次精准攻击，用于刷取"天鹰艺术家"等相关成就。

### 目录结构

```
任务流程/天鹰火炮成就/
├── __init__.py           # 任务链入口（刷天鹰火炮任务）
├── 天鹰火炮检测器.py      # YOLO v8 ONNX检测器
├── 天鹰火炮进攻任务.py    # 搜索+检测+攻击逻辑
├── 等待战斗结束并回营.py  # 放弃战斗回营
└── 模型/
    └── best.pt           # YOLO模型（首次运行自动转ONNX）
```

### 任务流程

1. 打开进攻页面（复用主世界打鱼任务）
2. 循环搜索敌人（最多50次）
3. 使用YOLO v8检测天鹰火炮位置
4. 选中雷电法术，攻击11次（±5像素抖动，模拟人类）
5. 放弃战斗并返回营地

### 使用方法

**GUI方式**：
在配置管理中将"是否刷天鹰火炮"设置为"开启"

**命令行方式**：
```bash
python 主入口.py 是否刷天鹰火炮
```

### 技术特点

- 独立的YOLO v8检测器（与主项目YOLO v5隔离）
- 自动模型转换（PT→ONNX，支持GPU/CPU）
- 小目标过滤（宽≥20, 高≥20, 面积≥400）
- 置信度阈值：0.20
- 攻击位置随机抖动（±5像素），模拟人类操作

## 主世界打鱼

**位置**：`任务流程/主世界打鱼/`

### 功能说明

自动搜索合适的村庄进行进攻，掠夺资源。

### 核心流程

1. 检测是否有可用工人
2. 打开进攻页面
3. 搜索村庄（检测资源和边缘比例）
4. 满足条件后下兵进攻
5. 等待战斗结束
6. 返回营地

### 配置项

- `欲进攻的最小资源` - 资源总量阈值
- `欲进攻资源建筑靠近地图边缘最小比例` - 边缘比例（0-1）
